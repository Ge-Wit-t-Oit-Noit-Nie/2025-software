Voor het ontwikkelen van de software wordt gebruik gemaakt van de {\bfseries{Nucleo-\/\+F412\+ZG Development}} board\hypertarget{index_autotoc_md27}{}\doxysection{\texorpdfstring{Middlewares}{Middlewares}}\label{index_autotoc_md27}
Voor het schrijven van de log op de SD card wordt gebruik gemaakt van een stukje code in {\ttfamily Middlewares/sd\+\_\+logger.\+c}. Deze code is betrekkelijk simpel. In het vervolg moet hier de logica zitten om de juiste info in de file te krijgen.

\begin{DoxyImportant}{Important}
Voor de {\ttfamily start\+Log\+Task} is {\itshape 3000 WORDS} gereserveerd op de stack. We moeten dus opletten met de code (en dus ook wat opschonen).
\end{DoxyImportant}
De functie {\ttfamily start\+Log\+Task} wacht op een item in de queue. Wanneer deze binnen komt, dan wordt er een string gemaakt. Deze string wordt naar de UART gestuurd en ook naar de SD\+\_\+\+LOGGER. Voor de sd\+\_\+logger wordt ook gebruik gemaakt van een MUTEX om te voorkomen dat er 2 schrijfacties tegelijk plaatsvinden.
